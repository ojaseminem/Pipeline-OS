# Vantadeck 1.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring the approved local-first Vantadeck roadmap to a release-candidate condition with honest platform and external-infrastructure gates.

**Architecture:** Desktop and CLI clients call the same Rust application service. Portable project state remains in `.vantadeck/project.toml`; machine-local state, caches, and operation history remain in SQLite. Platform discovery and VCS implementations stay behind traits, remote metadata is untrusted, and no manifest value is executed through a shell.

**Tech Stack:** Rust 1.96, Tokio, SQLx/SQLite, Tauri 2, React 19, TypeScript, Vite, JSON Schema, GitHub Actions.
