# Support

Use GitHub Discussions for usage questions and GitHub Issues for reproducible bugs.
Include the Vantadeck version, operating system, command or screen, expected result,
and sanitized logs. Never post credentials, private repository URLs, client names,
or proprietary project content.

Pre-alpha releases have no uptime or response-time guarantee. Paid support may be
offered independently but does not change public issue priority or unlock closed
official features.

Security vulnerabilities, leaked secrets, and artifact-integrity concerns are not
support requests. Report them privately as described in `SECURITY.md`; do not post
exploit details or suspect credentials in Discussions or Issues.

For release-installation problems, include the release tag, artifact filename,
SHA-256 verification result, operating-system version, CPU architecture, and whether
the platform signature or notarization check passed. The release evidence and known
external validation gates are documented under `docs/release/`.
